# DrivingLicenseSystem - მიმოხილვა

## ავტორი და კურსი
**ავტორი:** ირაკლი ზვიადაური  
**კურსი:** ინფორმაციული სისტემების ანალიზი და დაპროექტება (შუალედური გამოცდა)

## ამოცანის აღწერა
**მოკლე აღწერა:** სისტემის იმიტაცია მართვის მოწმობის გამოცდების ავტომატიზაციისთვის.  

**მთავარი მოთხოვნები:**  
- არსებობს ორი გამოცდა: **თეორია** და **პრაქტიკა**; თითოეულის მაქსიმალური ქულა 50.  
- თეორიის ჩაბარებისთვის მინიმუმ **15** ქულა. პრაქტიკის ჩაბარებისთვის მინიმუმ **20** ქულა.  
- თითო გამოცდაზე მაქსიმუმ **3** მცდელობა; მესამე წარუმატებლობის შემდეგ მომხმარებელი დაბლოკილია **1 წელი** (365 დღე).  
- მართვის მოწმობის მისაღებად საჭიროა ჯამში მინიმუმ **55** ქულა (თეორია + პრაქტიკა).  
- პრაქტიკის გამოცდაზე გასვლა შეუძლებელია თეორიის მინიმუმ 15 ქულის გარეშე.  
- სისტემა ინახავს მომხმარებელთა მონაცემებს (სახელი, პაროლი, ქულები, მცდელობები, მესამე წარუმატებლობის თარიღი, რეგისტრაციის და ბოლო აქტივობის თარიღები).

## გადაწყვეტა და არქიტექტურა
**მოკლე მიმოხილვა:**  
პროექტი არის კონსოლური Java აპლიკაცია, რომელიც მიჰყვება UML‑ში აღწერილ ლოგიკას. ძირითადი კომპონენტები:  
- **User კლასი** — ინახავს მომხმარებლის მონაცემებს და ლოგიკას დაბლოკვის შემოწმებისთვის.  
- **DrivingLicenseSystem კლასი** — პროგრამის მთავარი ლოჯიკა: ავტორიზაცია, რეგისტრაცია, გამოცდების ციკლი, მცდელობების და ქულების განახლება, დაბლოკვის მართვა.  
- **პერსისტენცია** — მომხმარებლები სერიულიზაციის საშუალებით ინახება ფაილში `users.dat`. პროგრამა იტვირთავს მონაცემებს სტარტზე და ინახავს ცვლილებებს რეგისტრაციის, გამოცდების და სისტემის დახურვის დროს.

**მთავარი გადაწყვეტილებები:**  
- ქულა ინახება როგორც საუკეთესო შედეგი; ახალი შედეგი შენახულია მხოლოდ თუ ის უფრო მაღალია.  
- მცდელობები ინკრემენტირდება თითო ჩაბარების მცდელობის შემდეგ; თუ მცდელობები მიაღწევს 3‑ს და მოთხოვნილი მინიმუმი არ შესრულდა, დგინდება `thirdFailDate`.  
- თუ `thirdFailDate` არსებობს, სისტემა არ აძლევს გამოცდაზე გასვლის უფლებას 365 დღის განმავლობაში; 365 დღის გასვლის შემდეგ მცდელობები და ბლოკი იწმინდება.

## ფაილები
- `src/main/java/ge/tbcacademy/DrivingLicenseSystem.java` — მთავარი პროგრამა.  
- `users.dat` — სერიული მონაცემების ფაილი (არ უნდა ჩატვირთოთ Git‑ში).
-

**შენახვის ადგილმდებარეობა:** `users.dat` შეიქმნება იმ დირექტორიაში, საიდანაც პროგრამა გაშვებულია. თუ გსურთ მონაცემების გაზიარება, შექმენით ნიმუშური ფაილი `users-sample.dat` ან გამოიყენეთ JSON ფორმატი.

5)	წვდომა ყველა ინფორმაციაზე (გარდა პირთა პირადი მონაცემებისა) რომელიც ეხება იმ პროცესებს რომლის ავტომატიზაციასაც გულისხმობს სისტემა.
